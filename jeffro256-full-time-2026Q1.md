## What

The last quarter was spent polishing the Carrot/FCMP++ integration, reviewing deeply, debugging, auditing, and upstreaming fixes. I implemented ArticMine's scaling changes, the unbiased hash-to-point changes needed in Carrot, and changes to Carrot that allows a post-quantum turnstile. I improved device support within the Carrot integration, and updated wallet knowledge proofs for FCMP++ & Carrot.

In the following quarter, I want to continue this work. Here is a list of tasks I want to work towards completing in rough order:

- Support/review wallet API changes by SNeedlewoods
- Integrate support for Carrot-derived accounts in wallet2
- Help launch FCMP++ Beta Stressnet with consensus / addressing protcol changes
- Further reach out and provide support to existing hardware wallet manufacturors on how to securely support Carrot/FCMP++ scanning/spending
- Solicit help for multisig implementations of Carrot
- Help out with organizing FCMP++ integration audits
- Review FCMP++ integration changes and otherwise help wherever I can

## Who

I have been contributing to the Monero core repository for [nearly four years](https://github.com/monero-project/monero/pulls?page=2&q=is%3Apr+author%3Ajeffro256) with a total of [141 merged commits to master](https://github.com/monero-project/monero/commits?author=jeffro256) thus far, with many open PRs. I wrote the [Carrot specification](https://github.com/jeffro256/carrot/blob/master/carrot.md), implemented it in [C++](https://github.com/monero-project/monero/pull/9559) and [Rust](https://github.com/jeffro256/carrot-rs), organized audits for both the [spec](https://github.com/cypherstack/carrot-audit) and [implementation](https://github.com/cypherstack/carrot_core-audit), and integrated it into the FCMP++ stressnet. I also worked on the Seraphis migration project in 2023/2024.
